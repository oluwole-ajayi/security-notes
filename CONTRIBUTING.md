# Contributing to security-notes

Thanks for considering a contribution.

This is a personal working notebook, so contributions aren't the primary function of the repository — but genuine improvements are welcome and merged where they fit the direction.

## Contributions that are especially welcome

- **Bug reports** on detection rules or IaC modules. Open an issue with the version you deployed, the environment, and what went wrong.
- **Detection rule improvements** — better tuning, additional false-positive filters, ATT&CK mapping corrections.
- **New detection rules** — please open an issue first to discuss scope. The catalogue aims to be focused rather than exhaustive.
- **Documentation improvements** — typos, unclear explanations, missing context.

## PR checklist

Before submitting a PR:

- Include a short description of what the change does and why.
- For detection rules: reference the ATT&CK technique(s) covered and include tuning notes.
- For IaC modules: confirm the module deploys cleanly against a fresh subscription or account, and include the deployment command in your PR description.
- Keep formatting consistent with existing content in the repository.

## Not in scope

- Marketing content — this repository is not a Techlync or VeriLync promotional channel.
- Product code from either of my ventures — those have their own repositories.
- Anything containing client-identifying information of any kind.
